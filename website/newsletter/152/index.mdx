## ⚛️ React {#react}

[![How React 18 Improves Application Performance](/emails/issues/152/react18perf.jpg)](https://vercel.com/blog/how-react-18-improves-application-performance)

**[How React 18 Improves Application Performance](https://vercel.com/blog/how-react-18-improves-application-performance)**

Comprehensive, well-illustrated article on React 18 and its impact on performance. Explains how Concurrent React works and its impact on metrics such as Total Blocking Time and Interaction to Next Paint. Also presents other new features such as the Transition API, React Server Components, Suspense and the new React cache.

---

[![Kuma UI](/emails/issues/152/kuma.jpg)](https://www.kuma-ui.com/)

**[Kuma UI](https://www.kuma-ui.com/)**

New CSS-in-JS library inspired by the DX offered by Chakra UI, and compatible with React Server Components. The library is based on a unique hybride method that combines the best of zero-runtime CSS-in-JS and traditional runtime solutions. We'll have to study all this, and see how this library stands up to [Panda CSS](https://panda-css.com/) from the Chakra team.

## 📱 React-Native {#react-native}

[![Expo SDK 49](/emails/issues/152/expo49.jpg)](https://blog.expo.dev/expo-sdk-49-c6d398cdf740)

**[Expo SDK 49](https://blog.expo.dev/expo-sdk-49-c6d398cdf740)**

The new SDK 49 has just been released. Many new features have been announced, including my favourites:

- React Native 0.72.3, React Native Web 0.19.6
- Expo Router v2.0
- Network debugging, VS Code debugging
- React devtools built-in
- Environment variables
- Local Expo Modules
- EAS Insights

**[Announcing Expo Router v2](https://blog.expo.dev/introducing-expo-router-v2-3850fd5c3ca1)**

Along with SDK 49, Expo is also announcing the release of Expo Router v2 with many enhancements, including:

- TypeScript support
- Static routes, SEO, Head metadata
- Async routes
- Universal Linking, handoff
- CSS and Styling
