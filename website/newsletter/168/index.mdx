---

- 💸 [From Dreamweaver to JSX: The Evolution of Visual Dev Tools for the Web](https://mightymeld.com/blog/browser-dev-tools-understand-code?utm_source=thisweekinreact&utm_medium=newsletter&utm_campaign=link1)
- 👀 [React Docs PR - React-DOM static resource loading functions](https://github.com/reactjs/react.dev/pull/6459)
- 👀 [Next.js PR - Next.js is exploring moving to Lightning CSS](https://github.com/vercel/next.js/pull/58712) 🐦 [Tweet](https://twitter.com/leeerob/status/1740124461683409042)
- 👀 [Remix PR - SPA Mode](https://github.com/remix-run/remix/pull/8338): Will add the long-awaited possibility to generate Single Page Applications with Remix.
- 👀 [Million.js joins Y Combinator](https://twitter.com/aidenybai/status/1737929698221682789): Their mission is “Make React Faster”.
- 🐦 [useMemo to become a better version of context selectors](https://twitter.com/sebastienlorber/status/1741218919732429108): We all want to optimize our apps with context selectors for a while, but it turns out [selectors do not compose well](https://twitter.com/dan_abramov2/status/1741200483862151663), and the plan is to improve useMemo to kind of act like a selector!
- 🐦 [React.postpone() experimental API](https://twitter.com/sebastienlorber/status/1741180119975182770)
- 🐦 [SkipRenderOnClient - interesting trick to optimize hydration performance](https://twitter.com/sebastienlorber/status/1742528219318738955)
- 📜 [How to make Storybook 2-4x faster - v7.6 comes with perf improvement options, enabled by default in v8.0](https://storybook.js.org/blog/optimize-storybook-7-6/)
- 📜 [Future of Storybook in 2024](https://storybook.js.org/blog/future-of-storybook-2024/): The themes for Storybook in 2024 will be application development (as opposed to isolated component development), enhanced support for non-React frameworks, and improved story generation (using AI).
- 📜 [MDX routes in Remix using Vite](https://remix-blog-mdx.fly.dev/): New approach to handle MDX at build-time using the new Vite support in Remix.
- 📜 [React Server Components: the Good, the Bad, and the Ugly](https://www.mayank.co/blog/react-server-components/): Critical, but well-informed take on the status quo of RSC.
- 📜 [Thoughts on Remix](https://www.viget.com/articles/thoughts-on-remix/): Balanced view on Remix - where it came from, their philosophy and how it compares to Next.js.
- 📜 [More Than You Need to Know About ReactDOM.­flushSync](https://julesblom.com/writing/flushsync): Detailed look at reac-dom’s flushSync method, a rarely used but sometimes crucial method to flush any updates inside a provided callback synchronously.
- 📜 [Next.js 14 Parallel Routes](https://www.builder.io/blog/nextjs-14-parallel-routes): Article explaining how to use parallel routes in Next.js to simultaneously render multiple pages within the same layout, as for example commonly required for dashboards.
- 📜 [Next.js 14 Intercepting Routes](https://www.builder.io/blog/nextjs-14-intercepting-routes): Introduction to “intercepting routes” in Next.js, which are most commonly used for login modals which should correctly be reflected in URL state.
- 📜 [React with Vite and TypeScript and its common challenges](https://wanago.io/2023/12/25/react-vite-typescript/)
- 📜 [What should we ship?](https://rauno.me/craft/vercel): Detailed blog post about the vercel.com relaunch.
- 📜 [Apollo Client 3.9 Feature Spotlight – The memory story](https://www.apollographql.com/blog/apollo-3-9-beta-feature-spotlight-the-memory-story)
- 📜 [Developing Gatsby: Enhancing stability and reliability](https://www.netlify.com/blog/developing-gatsby-enhancing-stability-and-reliability/)
- 📜 [The View Transition API: A New Way to Animate Page Transitions](https://marmelab.com/blog/2023/12/21/view-transitions.html)
- 📜 [Understanding React’s useFormState and useFormStatus Hooks](https://blog.logrocket.com/understanding-react-useformstate-useformstatus-hooks/): Currently, handling forms in React is mostly done with heavy manual lifting (think lots of useState hooks) or by leaning on a form library (like react-hook-form or Formik). Recent canary releases of React include two new hooks as kind of a middle ground between those existing approaches.
- 📜 [Diving into Server Actions in Next.js 14](https://blog.logrocket.com/diving-into-server-actions-next-js-14/): If you’re new to React Server Components/Next.js Server Actions, this is a good introductory article to catch up on what they’re all about.
- 📦 [shadcn-ui 0.5](https://github.com/shadcn-ui/ui/releases/tag/shadcn-ui%400.5.0): Adds a bunch of new components like Drawer, Carousel, and Pagination. 🐦 [Tweet](https://twitter.com/shadcn/status/1738283281424982488)
- [https://ui.shadcn.com/docs/changelog](https://ui.shadcn.com/docs/changelog)
- 📦 [rsc-cache - cache expensive RSCs, with Next.js integration](https://github.com/fredkiss3/rsc-cache)
- 📦 [TanStack Store - Framework agnostic, type-safe store with reactive framework adapters](https://tanstack.com/store/latest)
- 📦 [TanStack Config - Tool for publishing and maintaining high-quality JavaScript packages](https://github.com/TanStack/config)
- 📦 [Catalyst - Modern UI kit for React](https://tailwindcss.com/blog/introducing-catalyst): Tailwind UI, the paid component and template library from the makers of Tailwind CSS added a new UI kit.
- 📦 [Tokenami Atomic CSS in the style attribute](https://github.com/tokenami/tokenami): 2023 was the year of the styling libraries, and here is yet another one, which looks very well thought-out. Type-safe static styles with theming, responsive variant support, and no bundler integration.
- 📦 [StyleX 0.4.1](https://stylexjs.com/blog/v0.4.1): Meta’s StyleX keeps evolving, recently improved the ESLint and Rollup plugins, added auto-complete and performance and code-size improvements.
- 📦 [Storybook 8.0.0 alpha 4 - no react peer dependency anymore](https://github.com/storybookjs/storybook/releases/tag/v8.0.0-alpha.4)
- 📦 [TWC](https://react-twc.vercel.app/): library to create reusable Tailwind components for React
- 📦 [MDXTS](https://www.mdxts.dev/): Interesting approach to automatically render prose content, type documentation, and examples from MDX and TypeScript files (e.g. to generate documentation for React components). 🐦 [Tweet](https://twitter.com/souporserious/status/1738878942260699231)
- 📦 [Plate - Rich-text editor framework for React, customizable, including a plugin system](https://platejs.org/)
- 📦 [Holy-loader - Progress-bar for React and Next.js](https://github.com/tomcru/holy-loader)
- 📦 [next-route-handler-pipe - Function pipelines for Next.js route handlers](https://github.com/KolbySisk/next-route-handler-pipe)
- 📦 [puck 0.13 - Visual editor for React](https://github.com/measuredco/puck)
