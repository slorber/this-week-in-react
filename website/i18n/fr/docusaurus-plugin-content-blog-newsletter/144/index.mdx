## ⚛️ React {#react}

[![Introducing storage on Vercel](/emails/issues/144/vercel.jpg)](http://web.start-ui.com?utm_source=newsletter&utm_term=this-week-in-react)

**[Introducing storage on Vercel](https://vercel.com/blog/vercel-storage)**

Vercel vient de lancer son offre storage edge-first, particulièrement adaptée pour un usage dans les React Server Components et Next.js:

- Vercel KV: store key-value Redis, basé sur Upstash.
- Vercel Postgres: database PostgreSQL, basée sur Neon.
- Vercel Blob: file storage, basé sur Cloudflare R2.

C'est la [Vercel Ship week](https://vercel.com/ship): d'autres nouveautés devraient être annoncées dans les prochains jours. L'image choisie nous laisse penser qu'il y aura une release Next.js demain, introduisant une nouvelle feature **Next.js Server Actions** permettent de co-localiser une API server avec un form React. Cela dévoile aussi la volonté d'améliorer le [progressive enhancement](https://twitter.com/sebastienlorber/status/1653701538336915456) dans les apps React, d'une manière assez similaire à Remix: le formulaire peut aussi fonctionner sans JavaScript ou hydratation.

Note: cet exemple n'est pas vulnérable aux injections SQL car il utilise des template literals.
