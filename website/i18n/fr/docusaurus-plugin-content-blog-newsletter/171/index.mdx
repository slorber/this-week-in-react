## 📱 React-Native {#react-native}

Les actualités React Native de cette semaine sont compilées par Benedikt. Vous avez des commentaires, des suggestions ou des nouvelles à partager ? Faites-lui savoir sur [Twitter](https://twitter.com/bndkt).

[![Expo SDK 50](/emails/issues/171/expo50.jpg)](https://expo.dev/changelog/2024/01-18-sdk-50k)

**[Expo SDK 50](https://expo.dev/changelog/2024/01-18-sdk-50)**

Expo SDK 50 introduit les plugins Expo Dev Tools, vient avec de nouvelles API SQLite et Camera, et ajoute `@expo/fingerprint`, ainsi qu'une nouvelle façon de déterminer si une mise à jour du bundle JS est compatible avec un build natif donné. Cette version marque également la [fin du support de Webpack](https://blog.expo.dev/webpack-support-in-expo-cli-is-now-deprecated-e9831d7eb631) dans Expo, qui est maintenant déprécié en faveur de Metro. La meilleure manière de se familiariser avec Expo 50 est probablement de regarder Simon la parcourir en 15 minutes sur [YouTube](https://www.youtube.com/watch?v=shaGaiwu4Fg).

[![Expo Router v3](/emails/issues/171/router3.jpg)](https://expo.dev/changelog/2024/01-23-router-3)

**[Expo Router v3](https://expo.dev/changelog/2024/01-23-router-3)**

En même temps qu'Expo SDK 50, Expo Router v3 a été publié. Elle apporte de nombreuses améliorations en termes de performances et de stabilité, ainsi qu'une meilleure documentation. Plus particulièrement, il y a maintenant un support expérimental pour créer des endpoints serveurs universels avec API Routes. Expo Router est désormais le tout premier framework React full-stack universel !
