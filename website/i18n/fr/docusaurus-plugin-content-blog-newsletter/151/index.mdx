[![RBreaking Up with SVG-in-JS in 2023](/emails/issues/151/svginjs.jpg)](https://kurtextrem.de/posts/svg-in-js)

**[Breaking Up with SVG-in-JS in 2023](https://kurtextrem.de/posts/svg-in-js)**

Article très complet sur les diverses manières d'utiliser des SVGs dans une app React, et l'impacte sur les performances.
Déconseille le "SVG-in-JS": il faut éviter de render les SVGs via des composants React et d'avoir le SVG présent dans le bundle JS, y compris via des solutions type loader SVGR. Sauf exceptions, il vaut mieux éviter d'inliner les SVGs dans le HTML: les React Server Components ne sont pas forcément une solution idéale non plus.

[![Reinventing Camera Processing - TensorFlow Lite GPU straight from JavaScript](/emails/issues/151/visioncamera.jpg)](https://mrousavy.com/blog/Reinventing-Camera-Processing)

**[Reinventing Camera Processing - TensorFlow Lite GPU straight from JavaScript](https://mrousavy.com/blog/Reinventing-Camera-Processing)**

Marc Rousavy détaille comment il est capable de faire tourner des modèles TensorFlow sur React-Native Vision Camera v3 ([demo](https://twitter.com/mrousavy/status/1674791891823648770)). Son plugin vient juste d'être open-sourcé avec la sortie de [VisionCamera v3 rc.3](https://twitter.com/mrousavy/status/1675883155822202880). L'article est assez technique, car il explique le fonctionnement interne pour que cela tourne de manière performance a 60+ FPS. Il faut surtout retenir qu'il devient très facile d'intégrer un modèle TensorFlow à Vision Camera, et d'afficher un feedback en temps réel via React-Native-Skia. Il n'y a pas besoin d'écrire du code natif, et toute cette intégration peut même être modifiée via des updates Over-The-Air!
